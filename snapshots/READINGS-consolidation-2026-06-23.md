   - 占い結果履歴は `history:<userId>` の **単一パターンのみ**（重複・冗長・別系統テーブルなし）。
   - 各履歴要素: `{ action, result, createdAt, extra }`、新しい順・最大 30 件・TTL なし永続。

## 見送ったこと（と理由）

  データ移行を伴い、**データ損失・既存 API 破壊のリスクのみが増える**ため見送り（注意事項に合致）。
- KV キープレフィックスのリネーム/再編: 既存データのキー変更は破壊的なため見送り。
